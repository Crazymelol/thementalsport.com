# Paywalls

## Overview
A good paywall appears after the user feels value, makes the offer obvious, and removes risk. Convert on demonstrated value, not on blocking access prematurely.

## Framework
1. **Choose the model** — hard (gate everything), soft/freemium (gate premium features), or metered (N free then pay). Match to how value compounds.
2. **Time the gate** — trigger at a high-intent moment after an aha — not on first click.
3. **Design the screen** — lead with the benefit/outcome, show the plan (anchor annual vs monthly), highlight one recommended option, list what they unlock.
4. **Reduce risk** — free trial, money-back guarantee, no-card trial, clear cancel-anytime copy, transparent renewal terms.
5. **Reduce friction** — wallets/Apple Pay, prefilled details, minimal steps.
6. **Test** placement, trial length, copy, and price framing.

## Quick Reference
| Model | Best for | Risk |
|-------|----------|------|
| Hard | Clear premium value | High bounce if too early |
| Freemium | Network/habit products | Low conversion if free too good |
| Metered | Content/media | Easy to game |
| Free trial | Tools needing hands-on value | Card-required friction |

## Common Mistakes
- Gating before the user experiences value.
- Hiding renewal terms / surprise charges → chargebacks, distrust.
- Dark patterns making cancellation hard (regulatory + trust risk).
- One plan, no anchor, no recommended option.
- Card-required trials when no-card converts better long-term.
- Generic copy listing features instead of outcomes.
- No A/B testing of trigger point and pricing.
