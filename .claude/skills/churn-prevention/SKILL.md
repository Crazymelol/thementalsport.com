# Churn Prevention

## Overview
Most churn is predictable and preventable. Detect declining engagement early, intervene before cancel intent, and recover failed payments — retention compounds far more than acquisition.

## Framework
1. **Instrument signals** — login frequency, feature usage, declining engagement, support tickets, downgrade. Build an at-risk score.
2. **Proactive intervention** — re-onboard, nudge to the aha feature, check in before renewal for low-usage accounts.
3. **Cancel-flow saves** — ask reason, then offer a targeted save: pause, discount, downgrade, or feature help matching the reason. Don't trap them.
4. **Win-back** — re-engagement emails and offers to recently churned users.
5. **Fight involuntary churn (dunning)** — retry failed cards smartly, pre-expiry card-update prompts, grace periods, dunning emails. This is often 20-40% of churn and the easiest to recover.
6. **Close the loop** — analyze cancel reasons to fix root-cause product/onboarding issues.

## Quick Reference
| Churn type | Driver | Lever |
|-----------|--------|-------|
| Voluntary | Low value/usage | Onboarding, save offers |
| Involuntary | Failed payment | Dunning, card updater, retries |
| Active cancel | Price/missing feature | Pause, downgrade, discount |
| Silent | Disengagement | Usage nudges, re-onboarding |

## Common Mistakes
- Ignoring involuntary/payment churn (huge, easy recovery).
- Only reacting at the cancel button instead of early signals.
- Cancel flows that frustrate (dark patterns) and damage brand.
- Generic blanket discounts instead of reason-matched saves.
- No exit survey, so root causes stay invisible.
- Treating all churners the same vs segmenting by reason/value.
- Measuring only logo churn, ignoring revenue/net churn.
