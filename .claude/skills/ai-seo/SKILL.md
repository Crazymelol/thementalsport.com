# AI SEO (GEO / Answer Engine Optimization)

## Overview
LLMs cite content that is easy to extract, attributable, and entity-clear. Optimize for being quotable in a single passage, not just ranking on a SERP.

## Framework
1. **Answer-first structure** — lead each section with a direct, self-contained answer, then elaborate. LLMs lift standalone sentences.
2. **Question-shaped headings** — use H2/H3 that match natural-language queries ("How does X work?").
3. **Citability** — include concrete stats, dates, named sources, and quotable one-liners. Add original data; LLMs favor unique facts.
4. **Entity clarity** — define who/what plainly, link to authoritative sources, keep naming consistent so the model maps your brand to the entity.
5. **Machine-readable** — add FAQ/Article JSON-LD, clean semantic HTML, tables, and lists.
6. **Authority signals** — author bylines, credentials, citations, freshness (updated dates).

## Quick Reference
| Lever | Action |
|-------|--------|
| Extractability | Short, complete sentences; one claim each |
| Structure | Q&A headings, lists, tables, TL;DR |
| Trust | Sources, author bio, E-E-A-T, dates |
| Entity | Consistent naming, Wikipedia/Wikidata presence, schema |
| Coverage | Answer the full question + adjacent follow-ups |

## Common Mistakes
- Burying the answer under intro fluff — LLMs grab the first clear passage.
- Vague, unquotable prose with no concrete facts or numbers.
- Inconsistent brand/entity naming confusing the model.
- No structured data or semantic headings.
- Walls of text instead of scannable lists/tables.
- Ignoring follow-up questions users actually ask the AI.
- Chasing keyword density instead of genuine, citable expertise.
