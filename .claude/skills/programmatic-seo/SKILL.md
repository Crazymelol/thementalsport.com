# Programmatic SEO

## Overview
Combine a repeatable template with a quality dataset to target many long-tail queries at once. Each page must answer a real query with genuinely unique value, or Google deems it thin/doorway content.

## Framework
1. **Find a head term + modifier** with search demand and a natural variable axis (city, use-case, tool, persona).
2. **Source a real dataset** — proprietary or aggregated data is the moat. Each row must produce distinct, useful content.
3. **Build the template** with stable structure but high unique-content ratio per page (data tables, localized facts, real reviews, dynamic stats).
4. **Add unique value per page** beyond swapped keywords — pricing, availability, comparisons, user-generated content.
5. **Internal linking + sitemap** to help discovery; cluster pages by hub.
6. **Quality gate** — suppress/noindex pages with insufficient data; ship in batches and watch indexing.

## Quick Reference
| Pattern | Variable | Data needed |
|---------|----------|-------------|
| [service] in [city] | location | local listings, prices |
| best [X] for [persona] | use-case | curated rankings, specs |
| [A] vs [B] | pairs | feature/spec comparison |
| [tool] integrations | integration | per-tool details |

## Common Mistakes
- Spinning identical pages with only the keyword swapped → thin/doorway penalty.
- Publishing thousands at once instead of batching and monitoring indexing.
- Auto-generating pages for rows with no real data (empty templates).
- No internal linking, so pages never get crawled.
- Ignoring search intent — building pages no one searches.
- Letting low-value pages dilute crawl budget instead of noindexing them.
- No human spot-checks for accuracy and readability.
